## 3. TECHNICAL SAFEGUARDS (MANDATORY — follow exactly)

**3D & WebGL Integration:**
- Use the Three.js ecosystem via `@react-three/fiber` and `@react-three/drei`. No CSS-simulated 3D, no static images pretending to be 3D.
- Build the network background using `Points`/`InstancedMesh` for nodes and animated `Line`/`shaderMaterial` for connections — not a single default sphere with `MeshDistortMaterial`.
- Use `ShaderMaterial` or `drei`'s line glow techniques so the connection lines have real luminosity, not flat color.
- Lighting: `ambientLight` at low intensity plus one or two `pointLight`s near active nodes for depth. Use `Environment` from drei only if it doesn't wash out the dark background.
- Interaction: the network MUST respond to mouse movement — camera parallax or node drift lerping toward cursor position, subtle, not disorienting.

**Layout & Z-Index:**
- `<Canvas>` is `fixed inset-0 z-[-1]` (or `z-0` with content above).
- All content sections use `relative z-10` (or higher) and `bg-transparent`, so the WebGL background is visible through them.
- Only the footer/contact section may use a solid panel background if needed for legibility — everything else stays transparent.

**Typography & Contrast:**
- All text must pass WCAG AA contrast against the dark background at whatever opacity it's shown at. Test the hero headline and body copy specifically since they'll sit directly over the moving background.

**Performance:**
- Wrap all 3D content in `<Suspense>` with a lightweight fallback (no spinner over a dark background — use a simple fade-in).
- Cap node/particle count so the scene stays smooth on mid-range mobile (target: no visible frame drop on a mid-tier Android phone).
- Pause or simplify the animation when the tab is not visible (`document.visibilitychange`).
- Respect `prefers-reduced-motion`: if set, freeze the network to a static state and disable camera parallax.
